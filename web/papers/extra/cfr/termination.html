Termination experiments

We analysed 3 benchmark sets with different combinations of properties.

Summary

For termination analysis, we have experimentally evaluated the effect of Algorithm 1 on iRankFinder using 3 sets of standard benchmarks:
  • (Set-1) is a set of 416 ITSs coming from Java programs (directory From_AProVE_2014 of TPDB);
  • (Set-2) is a set of 806 ITSs coming not necessarily from structured programs (directory From_T2 of TPDB); and
  • (Set-3) is a set of 200 ITSs coming mainly from Java and C programs used by (Flores-Montoya 2017) for experimental evaluation of cost analysis.
Set-1 (resp. Set-2) includes 159 (resp. 348) ITS known to be non-terminating ITS, and thus we exclude them from the experiments. We first analysed all benchmarks for termination without CFR in two settings, using LRFs and using LLRFs, and for those on which iRankFinder failed, we analysed them again with CFR using different schemes and properties. There are 3 (vertical) blocks, each corresponding to a set of benchmarks. On the side of each block, we indicate the total number of benchmarks on which iRankFinder failed without CFR and the total time (in minutes) it took to analyze them, for both LRFs and LLRFs.

For each set, there are two (horizontal) blocks corresponding to the two settings. Columns correspond to different CFR schemes, and rows to set of properties used for CFR. These sets are completely defined on the paper but here you can find a summary:

  • P1 = {PRc}
  • P2 = {PRh }
  • P3 = {PRc ,PRh }
  • P4 = {PRc ,PRcv ,PRh ,PRhv }
  • P5 = {PRdh }
  • P6 = {PRdh ,PRc }
  • P7 = {PRdh ,PRh }

Each entry in the table includes the number of benchmarks that iRankFinder can now prove terminating and the total time it took to analyze all benchmarks, using the scheme and properties as indicated in the corresponding row and column. Each benchmark was given a 5-min time limit.

    In terms of precision:
  • For Set-1: without using CFR we prove termination of 159 (resp. 229) ITSs when using LRFs (resp. LLRFs), and with CFR we can handle 40 (resp. 9) more ITSs.
  • For Set-2: without using CFR we prove termination of 120 (resp. 376) ITSs when using LRFs (resp. LLRFs). With CFR we can handle 178 (resp. 82) more ITSs.
  • For Set-3: without using CFR we prove termination of 88 (resp. 175) ITSs when using LRFs (resp. LLRFs). With CFR we can handle 33 (resp. 13) more ITSs.

Times in the table should be interpreted carefully, since when analysing without CFR and when using CFRB the analysis stops as soon as it fails to prove termination of a SCC, while when using CFRS and CFRA it keeps trying to prove termination of all SCCs.

For cost analysis, we have applied the tool Koat to estimate the effect of CFR on precision. We have analysed set Set-3 without and with CFR with different properties. CFR improved (by inferring a tighter upper-bound) the precision in 15 benchmarks.
